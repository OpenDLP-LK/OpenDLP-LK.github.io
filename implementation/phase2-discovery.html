Phase 2: Discovery

Data Inventory & Classification | Months 2-4

Phase Overview

The Discovery phase focuses on identifying and classifying all sensitive data across your organization. This comprehensive inventory ensures your DLP solution protects the right data in the right ways, meeting PDPA's data processing transparency requirements.

Duration

2 months

Key Stakeholders

IT, Data Owners, Business Units

Success Criteria

Complete data inventory, classification schema

Key Objectives

1

Map Data Landscape

Identify all locations where sensitive data resides: databases, file shares, cloud storage, endpoints, email, collaboration tools.

2

Define Classification Schema

Create data classification framework (Public, Internal, Confidential, Restricted) aligned with PDPA sensitivity levels.

3

Inventory Sensitive Data Types

Catalog personal data (NIC, names, addresses), financial data (bank accounts), health records, and intellectual property.

4

Identify Data Flows

Map how data moves: internal transfers, external sharing, cloud uploads, email transmission, mobile device sync.

5

Assess Data Risk

Evaluate exposure risks, access controls, encryption status, and compliance gaps for each data type and location.

Data Classification Framework

Level 0 Public

Information approved for public disclosure with no restrictions.

Examples: Marketing materials, press releases, public website content

Level 1 Internal

Information for internal use only, not for external disclosure.

Examples: Internal policies, employee directories, operational procedures

Level 2 Confidential

Sensitive information requiring protection. Unauthorized access may cause moderate harm.

Examples: Financial statements, contracts, employee performance data, business plans

Level 3 Restricted

Highly sensitive data protected by PDPA. Unauthorized access may cause severe harm.

Examples: NIC numbers, bank accounts, health records, biometric data, trade secrets

Sri Lankan Data Patterns

Configure DLP detection for Sri Lankan-specific sensitive data patterns:

National Identity Card (NIC)

Old Format: 9 digits + V (e.g., 923456789V)

New Format: 12 digits (e.g., 199212345678)

Passport Numbers

Format: N followed by 7 digits (e.g., N1234567)

Bank Account Numbers

Varies by bank (typically 10-16 digits)

Include SWIFT codes for international transfers

Mobile Numbers

Format: +94 followed by 9 digits

Example: +94 77 123 4567

Discovery Tools & Methods

Automated Scanning

Use DLP discovery tools to scan file shares, databases, SharePoint sites, and cloud storage for sensitive data patterns.

  • • Regular expression matching
  • • Keyword searches
  • • Machine learning classification

Data Owner Interviews

Engage business unit leaders to identify critical data assets, understand workflows, and document handling requirements.

  • • Department interviews
  • • Process mapping sessions
  • • Risk assessments

Manual Audits

Review high-risk areas manually: HR files, finance records, customer databases, executive communications.

  • • Sample file reviews
  • • Email audits
  • • Access log analysis

Key Deliverables

Data Inventory Report

Complete catalog of sensitive data locations, types, volumes, and risk levels.

Data Classification Schema

Approved framework with clear definitions, examples, and handling requirements.

Data Flow Diagrams

Visual maps showing how sensitive data moves within and outside the organization.

Risk Assessment

Prioritized list of data protection risks with recommendations for Phase 3 policy development.

Best Practices

Start with High-Risk Areas

Prioritize HR, Finance, Customer Service departments where sensitive data concentration is highest.

Test Detection Patterns

Validate that your NIC, passport, and bank account patterns work correctly before full deployment. Test with sample data.

Document Data Owners

PDPA requires clear data ownership. Assign responsible parties for each data category discovered.

Address Shadow IT

Don't forget unsanctioned cloud services (personal Dropbox, Gmail) where employees may store company data.
